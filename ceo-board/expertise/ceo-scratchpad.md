# CEO Strategic Scratchpad

## 2026-06-01 — Atlas return growth pattern
For Atlas, board consensus remains: do not increase exposure first when live approval=false and research trust gates are open. Return-improvement sequencing should be: (1) regression harness + stale-test cleanup + post-repair baselines, (2) paper/backtested sizing improvements, (3) one additive OOS/correlation-gated strategy, (4) only then live config/exposure decisions. A small research-only new-project spike is acceptable if it creates no infrastructure/capital obligation and auto-kills when Atlas critical path slips.
